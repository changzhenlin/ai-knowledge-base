# Java RAG / Agent 面试题（中级） - 2026-03-24

## 题目
Java 项目里做 RAG 时，为什么文档切分（chunking）策略会直接影响召回效果？你会怎么设计？

## 详解答案
因为向量检索本质上是“拿 query embedding 去匹配 chunk embedding”，如果 chunk 切得过大，语义太杂；切得过小，上下文又不完整，都会导致召回结果失真。实际设计时，我会优先按语义边界切分，比如标题、段落、代码块、表格分别处理；再结合 token 长度设置 chunk size 和 overlap。对于 FAQ/制度文档适合较小 chunk，对于技术方案、接口文档更适合结构化切分。最后会通过 recall@k、人工抽样评估来迭代。

## 面试官真正想听什么
- chunk 过大/过小都会影响召回
- 按语义边界切分优于纯长度切分
- 用 recall@k 和人工抽样持续优化
