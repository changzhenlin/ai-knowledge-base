# Java RAG / Agent 面试题（中级） - 2026-03-23

## 题目
用 Java 做 RAG 检索链路时，你会如何设计一个更可靠的 retrieval pipeline？

## 详解答案
我一般不会只做一次向量检索，而是把 retrieval pipeline 设计成“查询预处理 + 多路召回 + 重排 + 上下文组装”。查询预处理包括 query rewrite、关键词提取、同义词扩展；召回层可以是 BM25 + 向量检索 + 标签过滤；重排层用 reranker 或基于业务规则的排序；最后再按 token 预算组装上下文。这样做的原因是，单路向量检索在专业术语、数字、版本号、实体名上经常翻车，多路召回更稳。

## 面试官真正想听什么
- 多路召回比单路向量检索更稳
- 重排是提升答案质量的关键
- 最终上下文组装要考虑 token 预算
